# Chrome Web Store 제출 가이드

## 📦 업로드 파일
**파일명**: `webguard-qa-chrome-store.zip` (25KB)
**내용**: extension 폴더의 모든 파일

## 🚀 제출 절차

### 1단계: Chrome Web Store 개발자 계정 생성
1. https://chrome.google.com/webstore/devconsole 접속
2. Google 계정으로 로그인
3. **일회성 등록비 $5 결제** (신용카드 필요)
4. 개발자 정보 입력

### 2단계: 새 항목 추가
1. 개발자 대시보드에서 "새 항목" 클릭
2. `webguard-qa-chrome-store.zip` 업로드
3. 자동 검증 대기 (1-2분)

### 3단계: 스토어 등록 정보 작성

#### 필수 정보

```
웹사이트의 보안 취약점을 자동 탐지하고 분석하는 통합 QA 도구. XSS, CSRF, 보안 헤더 등을 실시간으로 검사합니다.
```

**상세 설명**:

### 4단계: 스크린샷 준비

**필수 스크린샷** (최소 1개, 최대 5개):
- 크기: 1280x800 또는 640x400
- 형식: PNG 또는 JPG

**권장 스크린샷**:
1. **팝업 화면** - 스캔 실행 중
2. **스캔 결과** - 발견된 문제 표시
3. **대시보드 개요** - 통계 화면
4. **취약점 분석** - 상세 내역
5. **도움말 페이지** - 기능 설명

### 5단계: 아이콘 준비

**필수 아이콘**:
- 128x128 PNG (스토어 표시용)

**현재 상태**: SVG 파일만 있음
**해결 방법**: 
1. https://cloudconvert.com/svg-to-png 접속
2. `extension/assets/icon.svg` 업로드
3. 128x128 크기로 변환
4. 다운로드하여 업로드

### 6단계: 개인정보처리방침

**필수**: 개인정보처리방침 URL

**권장 내용**:
```
WebGuard QA 개인정보처리방침

1. 수집하는 정보
WebGuard QA는 어떠한 개인정보도 수집하지 않습니다.

2. 데이터 저장
모든 스캔 결과는 사용자의 로컬 브라우저(Chrome Storage)에만 저장됩니다.
외부 서버로 전송되지 않습니다.

3. 권한 사용
- activeTab: 현재 탭에서 보안 스캔 수행
- storage: 스캔 결과 로컬 저장
- tabs: 대시보드 및 도움말 페이지 열기
- contextMenus: 우클릭 메뉴 통합
- notifications: 심각한 문제 발견 시 알림
- downloads: 결과 내보내기

4. 제3자 공유
사용자 데이터를 제3자와 공유하지 않습니다.

5. 문의
GitHub: [Your GitHub URL]
```

**호스팅 방법**:
- GitHub Pages: `https://yourusername.github.io/webguard-qa/privacy.html`
- 또는 Tistory 블로그에 게시

### 7단계: 심사 노트 (선택사항)

```
이 확장 프로그램은 웹 개발자를 위한 보안 테스트 도구입니다.
모든 기능은 로컬에서 실행되며, 외부 서버와 통신하지 않습니다.

테스트 방법:
1. 아무 웹사이트 방문 (예: https://example.com)
2. 확장 프로그램 아이콘 클릭
3. "빠른 스캔" 버튼 클릭
4. 결과 확인

주요 기능:
- 보안 취약점 자동 탐지
- 실시간 분석 및 리포팅
- 완전한 오프라인 작동
```

## ⚠️ 주의사항

### 필수 확인 사항:

- ❌ 스크린샷 누락
- ❌ 설명 부족
